CDN

Manuel olarak güncelleyebileceğiniz belirli bir sürüm numarasına bağlantı oluşturmanızı öneririz:

<script src= "https://cdn.jsdelivr.net/npm/vue@2.5.16/dist/vue.js" ><script>

NPM paketinin kaynağına cdn.jsdelivr.net/npm/vue adresinden göz atabilirsiniz.

Vue'nin farklı yapılarını okuduğunuzdan ve üretim versiyonunu kullandığınızdan emin olun. Yayımlanan sitenizdeki, vue.js dosyasını vue.min.js. ile değiştirin Bu, geliştirme deneyimi yerine hız için optimize edilmiş daha küçük bir yapıdır.

NPM

Vue ile büyük ölçekli uygulamalar oluştururken NPM, önerilen kurulum yöntemidir. Webpack veya Browserify gibi modül grupları ile güzelce eşleşir. Vue ayrıca Tek Dosya Bileşenlerinin yazılması için eşlik eden araçlar sağlar.

$ npm install vue

Farklı Yapıların Tanımlanması

NPM paketinin dist/ dizininde Vue.js.'nin birçok farklı yapısını bulacaksınız.

Aralarındaki farka bir bakalım :

UMD CommmonJS ES Modülü
Tam vue.js vue.common.js vue.esm.js
Runtime-only vue.runtime.js vue.runtime.common.js vue.runtime.esm.js
Tam(Üretim) vue.min.js - -
Runtime-only(Üretim) vue.runtime.min.js - -

Terimler

  • Tam: Hem derleyiciyi hem de çalışma zamanını içeren oluşturur.

  • Derleyici: Şablon dizelerini JavaScript oluşturma işlevlerine derlemekten sorumlu kod.

  • Runtime: Sanal DOM, vb. Oluşturma ve düzeltme, Vue örneklerini oluşturmadan sorumlu kod.

  • UMD : UMD yapıları doğrudan tarayıcıda bir script etiketi ile kullanılabilir. https://cdn.jsdelivr.net/npm/vue adresindeki jsDelivr CDN'deki varsayılan dosya, Runtime + Derleyici UMD yapısıdır (vue.js).

  • CommonJS : CommonJS yapıları, browserify veya webpack 1 gibi eski gruplarla kullanım için tasarlanmıştır. Bu grupların varsayılan dosyası (pkg.main), yalnızca Çalışma Zamanı CommonJS yapısıdır(vue.runtime.common.js).

  • ES Module : ES modül yapıları, webpack 2 ya da rollup gibi modern paketlerle kullanılmak üzere tasarlanmıştır. Bu paketleyiciler için varsayılan dosya (pkg.module), yalnızca Çalışma Zamanı CommonJS yapısıdır (vue.runtime.esm.js).

Runtime+Derleyici ve Runtime-only

İstemcide şablon derlemeniz gerekiyorsa, derleyiciye ve dolayısıyla tam yapısına ihtiyacınız olacak:

// Bu derleyici gerektirir
new vue({
template: '<div>{{ hi }}</div>'
})

// Bu gerektirmez
new vue({
render (h) {
return h('div', this.hi)
}
})

vue-loader veya vueify kullanırken, *.vue dosyalarındaki şablonlar derleme zamanında JavaScript'e derlenir. Derleyiciye son pakette gerçekten ihtiyacınız yok ve bu nedenle Runtime-only kullanabilirsiniz.

Runtime-only oluşturma özellikleri, tam olarak oluşturulmuş muadillerine göre kabaca %30 daha hafif olduklarından, bunu her fırsatta kullanmalısınız. Bunun yerine tam yapıyı kullanmak istiyorsanız, paketleyicinizde bir takma ad yapılandırmanız gerekir:

Webpack

module.exports = {
//...
resolve: {
alias: {
'vue$' : 'vue/dist/vue.esm.js' // Webpack 1 için 'vue/dist/vue.common.js'
}
}
}

Rollup

const alias = require( 'toplayicinin-takma-adi' )

rollup({
//...
plugins: [
alias({
'vue' 'vue/dist/vue.esm.js'
})
]
})

Browserify

package.json'ı projenize ekleyin:

{
//...
"tarayici" : {
"vue": "vue/dist/vue.common.js"
}
}

Parcel

package.json'ı projenize ekleyin:

{
//...
"alias" : {
"vue": "./node_modules/vue/dist/vue.common.js"
}
}

Geliştirme ve Üretim Modu

Geliştirme / üretim modları UMD için sabit kodlanmıştır: minimilize edilmemiş dosyalar geliştirme içindir ve minimilize edilmiş dosyalar üretim içindir.

CommonJS ve ES Modülü yapıları, paketleyiciler için tasarlanmıştır, bu nedenle bunlar için küçültülmüş sürümler sağlarız. Son paketi kendiniz küçültmekten siz sorumlu olacaksınız.

Webpack

Webpack 4+'da, mod seçeneği kullanabilirsiniz.

module.exports={
mode:'uretim'
}

Ama Webpack 3 ve daha önceki sürümlerinde, DefinePlugin'i kullanmanız gerekir:

var webpack = require( 'webpack')

module.exports = {
//...
plugins: [
//...
new webpack.DefinePlugin({
'process.env': {
NODE_ENV: JSON .stringify( 'uretim')
}
})
}
}

Rollup

rollup-plugin-replace kullanın.

const replace = require('rollup-plugin-replace)

rollup({
//...
plugins: [
replace({
'process.env.NODE_ENV': JSON.stringify('uretim')
})
]
}).then(...)

Browserify

Global envify dönüşümünü paketinize uygulayın.

NODE_ENV=production browserify -g envify -e main.js | uglifyjs -c -m > build.js

Ayrıca bkz. Üretim Dağıtım İpuçları.

CSP ortamları

Google Chrome Uygulamaları gibi bazı ortamlarda, içeriklerin değerlendirilmesi için yeni Function() işlevinin kullanılmasını yasaklayan İçerik Güvenliği Politikası (CSP) uygulanmaktadır. Tam yapı, şablonların derlenmesi için bu özelliğe bağlıdır, bu nedenle bu ortamlarda kullanılamaz.

Öte yandan, yalnızca çalışma zamanı yapısı tamamen CSP uyumludur. Webpack + vue-loader veya Browserify + vueify ile çalışma zamanı oluşturmayı kullanırken, şablonlarınız CSP ortamlarında mükemmel şekilde çalışan render işlevleri haline getirilecektir.

İşlem Tanımlama

Bu dersi Scrimba'da dene

Vue.js’in çekirdeği, basit şablon sözdizimini kullanarak DOM’daki verilere bildirimde bulunmamızı sağlayan bir sistemdir:

<div id="app">
{{ mesaj }}
</div>
var app = new Vue({
el: '#app' ,
data: {
mesaj: 'Merhaba Vue!'
}
})
{{ message }}

İlk Vue uygulamamızı oluşturduk! Bu, bir dize şablonu oluşturmaya benziyor, ancak vue başlık altında çok fazla iş yaptı. Veriler ve DOM artık birbirine bağlı ve herşey artık duyarlı.Nasıl bilebiliriz? Tarayıcınızın JavaScript konsolunu (hemen bu sayfada) açın ve app.message değerini farklı bir değere ayarlayın. Yukarıdaki güncellemeye göre hazırlanan örneği görmelisiniz.

Metnin ara değerlerine ek olarak, bunun gibi öğe özelliklerini de bağlayabiliriz:

<div id="app-2">
<span v-bind:title="message">
Fareyi birkaç sanyeliğine burada tutarsanız başlığı görebilirsiniz!
</span>
</div>
var app2 = new Vue ({
el: '#app-2',
data: {
message: 'Bu sayfayı + new Date().toLocaleString() + 'yüklediniz!'
}
})
Fareyi birkaç saniyeliğine burada tutarsanız başlığı görebilirsiniz!

Burada yeni bir şeyle karşılaşıyoruz. Gördüğünüz v-bind özelliğine direktif denir. Direktifler, Vue tarafından sağlanan özel nitelikler olduklarını belirtmek için v- ile öne eklenir ve tahmin edebileceğiniz gibi, oluşturulan DOM’a özel duyarlı davranışlar uygularlar. Burada temel olarak “Bu öğenin title (başlık) özelliğini Vue örneğindeki mesaj özelliği ile güncel tut” deyip duruyor.

JavaScript konsolunuzu tekrar açarsanız app2.message = 'yeni mesaj', bir kez daha bağlı HTML'nin - bu durumda title(başlık) özelliğinin - güncellendiğini göreceksiniz.

<div id="app-3">
<span v-if="seen">Şimdi Beni Görüyorsun </span>
</div>
var app3 = new Vue({
el: '#app-3',

}
})

Kullanıcı Girişini Kullanma

Bu dersi Scrimba'da dene

<div id="app-5">
<p>{{ message }} </p>
<button v-on:click="reverseMessage">Mesajı Ters Yaz</button>
</div>
var app5 = new Vue({
el: '#app-5',
data: {
message: 'Merhaba Vue.js'
},
methods: {
reverseMessage: function () {
this.message = this.message.split('').reverse().join('')
}
}
})

{{ message }}

Bu yöntemde, DOM'a dokunmadan uygulamanızın durumunu güncelleriz. Tüm DOM manipülasyonları Vue tarafından işlenir ve yazdığınız kod temel mantığa odaklanır.

Vue ayrıca, form girişi ve uygulama arasında iki yönlü bir bağlantı kuran v-model yönergesini de sağlar:

<div id="app-6">

<input v-model="message">
</div>

}
})

{{ message }}

Bileşenler Oluşturma

Bu dersi Scrimba'da dene

Bileşen sistemi, Vue'daki diğer önemli bir kavramdır, çünkü küçük, kendi kendine yeten ve çoğu zaman yeniden kullanılabilen bileşenlerden oluşan büyük ölçekli uygulamalar oluşturmamıza olanak veren bir soyutlamadır. Bunu düşünürsek, hemen hemen her türlü uygulama arayüzü, bileşen ağacına ayrılabilir:

Vue'da, bir bileşen esas olarak önceden tanımlanmış seçeneklere sahip bir Vue örneğidir. Bir bileşenin Vue'ya kaydedilmesi basittir:

//Todo-item adı verilen yeni bir bileşen tanımlayın
Vue.component('todo-item', {
template: '<li>Bu bir todo</li>'
})

Şimdi başka bir bileşenin şablonunda yazabilirsiniz:

<ol>
<!--todo-item bileşeni örneğin oluşturma -->
<todo-item></todo-item>
</ol>

Ancak bu her bir todo için aynı metni oluşturur, bu gayet normal. Ana kapsamdaki verileri çocuk bileşenlerine geçirebilmeliyiz. Bir prop kabul etmesini sağlamak için bileşen tanımını değiştirelim:

Vue.component('todo-item', {
//todo-item bileşeni artık prop'u bir özel özellik
// gibi kabul eder.
//Bu prop'a todo denir.
props: ['todo'],
template: '<li>{{ todo.text }}</li>'
})

Şimdi v-bind kullanarak her tekrarlanan bileşene todo geçiş yapabiliriz:

<div id="app-7">
<ol>
<!--
Şimdi her todo-item todo nesnesini temsil ediyor, böylece içeriği
Dinamik oluyor.Ayrıca her bir bileşeni bir “anahtar" ile
sağlamamız gerekiyor.
-->
<todo-item
v-for="item in groceryList"
v-bind:todo="item"
v-bind:key="item.id">
&/todo-item>
</ol>
</div>
Vue.component('todo-item', {
props: ['todo'],
template: '<li>{{ todo.text }}</li>'
})

var app7 = new Vue({
el: '#app-7',
data: {
groceryList: [
{ id: 0, text: 'Sebzeler' },
{ id: 1, text: 'Peynir' },
{ id: 2, text: 'İnsanlar başka ne yemeli?' },
]
}
})

Bu, zor bir örnektir, ancak uygulamamızı iki küçük bölüme ayırmayı başardık ve çocuk, props arabirimi aracılığıyla ebeveynden makul ölçüde iyi bir şekilde ayrıştırıldı.

Büyük bir uygulamada, tüm uygulamayı geliştirirken yönetimi kolaylaştırmak için bileşenlere bölmek gerekir. İleride bileşenler hakkında çok daha fazla konuşacağız, ancak bir uygulamanın şablonunun bileşenlerle neye benzeyebileceğne dair bir örnek:

<div id="app">
<app-nav></app-nav>
<app-view>
<app-sidebar></app-sidebar>
<app-content></app-content>
</app-view>
</div>

Vue Örneği

Bir Vue Örneği Oluşturmak

Her bir Vue uygulaması, Vue işlevine sahip yeni bir Vue örneği oluşturarak başlar:

var vm = new Vue({
//ayarlar
})

MVVM modeli ile kesinlikle ilişkili olmamasına rağmen, Vue’nin tasarımı kısmen esinlenmiştir. Bir kural olarak, Vue örneğimize başvurmak için genellikle vm değişkenini (ViewModel kısaltılması) kullanırız.

Bir Vue uygulaması, isteğe bağlı olarak yuvalanmış, yeniden kullanılabilir bileşenlerin bir ağacına düzenlenmiş yeni Vue ile oluşturulan bir kök Vue örneğinden oluşur. Örneğin, bir yapılacaklar uygulamasının bileşen ağacı şöyle görünebilir:

Ana Yapı
└─ YapılacaklarListesi
├─ Yapılacak İş
│ ├─ GöreviSilButonu

Veri ve Methodlar

Bir Vue örneği oluşturulduğunda, veri nesnesinde bulunan tüm özellikleri Vue’nin reaktivite sistemine ekler. Bu özelliklerin değerleri değiştiğinde, görünüm “yeni değerler” ile eşleşecek şekilde güncellenir.

//veri nesnemiz
var veri = { a: 1 }

// Nesne bir vue örneğine eklenir
var vm = new Vue({
veri: veri
})

// Örnekte özellik edinme
// orjinal verilerden birini döndürür
vm.a == veri.a // => true
// Örnekte özelliğin değiştirilmesi
// orjinal verileri de etkiler
vm.a = 2
data.a // => 2

//... ve tersi içinde geçerli.
data.a = 3
vm.a // => 3

Bu veriler değiştiğinde, görünüm yeniden oluşturulur. Verilerdeki özelliklerin, yalnızca örnek oluşturulduğunda mevcutsa reaktif olduklarına dikkat edilmelidir. Bu, aşağıdaki gibi yeni bir özellik eklerseniz:

vm.b = 'Merhaba'

Ardından b olarak yapılan değişiklikler, herhangi bir görünüm güncellemesini tetiklemez. Daha sonra bir özelliğe ihtiyacınız olacağını biliyorsanız, bazı başlangıç değerlerini ayarlamanız gerekir. Örneğin :

veri: {
yeniYapılacakİş: '',
görüntülenmeSayısı: 0,
tamamlanmışTodolarıGizle: false,
yapılacaklar: [],
error: null
}

Bunun tek istisnası, mevcut özelliklerin değiştirilmesini engelleyen Object.freeze() kullanımıdır, bu da reaktivite sisteminin değişiklikleri izleyemediği anlamına gelir.

var obj = {
foo: 'bar'
}

Object.freeze(obj)

new Vue({
el: '#app',
veri: obj
})
<div id="app">
<p>{{ foo }} </p>
<!-- Bu artık foo'yu güncelleyemez! -->
<button v-on:click="foo = 'baz'">Değiştir</button>
</div>

Veri özelliklerine ek olarak, Vue örnekleri bir dizi yararlı örnek özellik ve yöntemi açığa çıkarır. Bunlar, kullanıcı tanımlı özelliklerden ayırt etmek için $ ile öneklenir. Örneğin:

var data = { a: 1}
var vm = new Vue({
el: '#ornek',

})

vm.$veri === veri // => true
vm.$el === document.getElementById('ornek') // => true

// $izle bir örnek methodtur

// vm.a değiştiğinde bu geri arama çağrılır.
})

Örnek Akış Kancaları

Her bir Vue örneği, oluşturulduğunda bir dizi başlatma adımından geçer - örneğin, veri gözlemini kurması, şablonu derlemesi, örneği DOM'a bağlaması ve veri değiştiğinde DOM'ı güncellemesi gerekir. Ayrıca yaşam döngüsü kancaları olarak adlandırılan fonksiyonları çalıştırır ve yol boyunca kullanıcılara belirli aşamalarda kendi kodlarını ekleme fırsatı verir.

Örneğin oluşturulan kanca, bir örnek oluşturulduktan sonra kodu çalıştırmak için kullanılabilir:

new Vue({
veri: {
a: 1
},
olusturulan: function() {
// bu, vm örneğini işaret ediyor.
console.log('a =' + this.a)
}
})
// => "a = 1"

Eklenen, güncellenen ve yok edilen yaşam d öngüsü örneğinin farklı aşamalarında çağrılacak başka kancalar da vardır. Tüm yaşam döngüsü kancaları, bu bağlamı çağırmak için Vue örneğini işaret ederek bu içeriğe çağrılır.

Hedef fonksiyonlarını, bir ayarlar özelliğinde veya geri aramada, örneğin: created: () => console.log(this.a) veya vm.$watch('a', newValue => this.myMethod()) gibi fonksiyonlarda kullanmayın. Hedef fonksiyonları ana içeriiğe bağlı olduğunan, bu beklediğiniz gibi bir Vue örneği olmaz, genellikle Uncaught TypeError: Cannot read property of undefined ya da Uncaught TypeError: this.myMethod is not a function gibi hatalarla sonuçlanır.

Şablon Söz Dizimi

Vue.js, oluşturulmuş DOM'yi temel Vue örneğinin verilerine bildirmenize olanak tanıyan bir HTML tabanlı şablon sözdizimi kullanır. Tüm Vue.js şablonları, uyumlu tarayıcılar ve HTML ayrıştırıcıları tarafından ayrıştırılabilen geçerli bir HTML'dir.

Virtual DOM kavramlarına aşina iseniz ve JavaScript'in ham gücünü tercih ediyorsanız, isteğe bağlı JSX desteği ile şablonlar yerine render işlevlerini doğrudan yazabilirsiniz.

Ara Değer Bulma

Metin

En basit veri bağlama biçimi “Bıyık” sözdizimini (çift küme parantezleri) kullanarak metin enterpolasyonudur:

<span>Message: {{ msg }}</span>

Bıyık etiketi, ilgili veri nesnesi üzerindeki msg özelliğinin değeriyle değiştirilecektir. Ayrıca, veri nesnesinin msg özelliği değiştiğinde de güncellenecektir.

Aynı zamanda v-once direktifi kullanarak veri değişiminde güncellenmeyen bir kerelik ara değerleri de gerçekleştirebilirsiniz, ancak aynı alandaki diğer bağlamaları da etkileyeceğini unutmayın:

<span v-once>Bu asla değişmeyecek: {{ msg }}</span>

Ham HTML

Çift süslü parantez, verileri HTML değil düz metin olarak yorumlar. Gerçek HTML çıktısı almak için v-html yönergesini kullanmanız gerekir:

<p>Süslü parantez kullanma: {{ hamHTML }}</p>
<p>v-html direktifini kullanma: <span v-html="hamHTML"</p>

Süslü parantez kullanma: {{ hamHTML }}

v-html direktifi kullanma:

span içeriği, düz HTML olarak yorumlanan rawHtml özelliğinin değeriyle değiştirilir; veri bağlantıları yoksayılır. Şablon kısmi oluşturmak için v-html kullanamayacağınızı unutmayın, çünkü Vue, dize tabanlı bir şablon oluşturma motoru değildir.

Web sitenizde dinamik HTML'yi dinamik olarak oluşturmak çok tehlikeli olabilir çünkü XSS güvenlik açıklarına kolayca yol açabilir.Yalnızca güvenilir içerik üzerinde HTML interpolasyonunu kullanın ve asla kullanıcı tarafından sağlanan içerik üzerinde kullanmayın.

Özellikler

Süslü parantezler HTML özniteliklerinde kullanılamaz. Bunun yerine bir v-bind direktifini kullanın:

<div v-bind:id="dinamikId"></div>

Sadece varlıkların doğru olduğu yerlerde, boole özelliklerinde, v-bind biraz farklı çalışır. Bu örnekte:

<button v-bin:disabled="butonDevredisi">Button</div>

Eğer butonDevredisi öğesi null, undefined veya false değerine sahipse, disabled özelliği oluşturulmuş <button> öğesine dahil edilmez.

Javascript İfadelerini Kullanma

Şimdiye kadar sadece şablonlarımızdaki basit özellik anahtarlarına bağlanıyoruz. Ancak Vue.js aslında tüm veri bağlamaları içindeki JavaScript ifadelerinin tam gücünü desteklemektedir:

{{ numara + 1 }}

{{ tamam ? 'EVET' : 'HAYIR' }}

{{ message.split('').reverse().join('') }}

<div v-bind:id="'list-' + id"></div>

Şablon ifadeleri korumalı ve yalnızca Matematik ve Tarih gibi bir beyaz küre listesine erişebilir. Şablon ifadelerinde kullanıcı tanımlı globallere erişmeyi denememelisiniz.

Direktifler

Talimatlar v- ön eki ile ayrıcalıklı özelliklerdir. Talimat özellik değerlerinin tek bir JavaScript ifadesi olması beklenir (v-for için istisna, daha sonra tartışılacaktır).Bir direktörün görevi, ifade ettiği değer değiştiğinde, DOM’a yan etkileri tepkisel olarak uygulamaktır.Girişte gördüğümüz örneği inceleyelim:

<p v-if="gor">Şimdi beni görüyorsun.</p>

Burada, v-if yönergesi gor ifadesinin değerinin doğruluğuna dayanarak <p> öğesini kaldıracak / ekleyecektir.

Argümanlar

Bazı direktifler "argüman" alabilirler ve direktif adından sonra bir kolon tarafndan gösterilirler. Örneğin, v-bind yönergesi bir HTML özniteliğini yeniden etkinleştirmek için kullanılır:

<a v-bind:href="url">...</a>

Burada href, v-bind yönergesinin, öğenin href özniteliğini ifade url'sinin değerine bağladığını söyleyen argümandır.

Bu örnek ise DOM olaylarını dinleyen v-on yönergesidir:

<a v-on:click="birŞeyYap">...</a>

Düzenleyiciler

Değiştiriciler, bir nokta ile belirtilen özel postfix'lerdir. Bir direktifin bazı özel yollarla bağlanması gerektiğini belirtir.Örneğin, .prevent değiştiricisi, tetiklenen olayla ilgili v-on yönergesine event.preventDefault() öğesini çağırmasını söyler:

<form v-on:submit.prevent="Gonder">...</form>

Kısayollar

v- ön eki, şablonlarınızdaki Vue'ya özgü öznitelikleri tanımlamak için görsel bir işaret görevi görür. Bu, varolan bazı biçimlendirmelere dinamik davranış uygulamak için Vue.js'yi kullanırken yararlıdır. ama sıkça kullanılan bazı direktifler için ayrıntılı olabilir. Aynı zamanda Vue.js'nin her şablonu yönetdiği bir SPA oluştururken v- öneki gereksinimi daha az önem kazanır. Bu nedenle, Vue.js en sık kullanılan direktiflerden ikisi, v-bind ve v-on için özel kısayollar sağlar:

v-bind Kısayolu

<!-- tam yazılışı -->
<a v-bind:href="url">...</a>

<!-- kısayolu -->
<a :href="url">...</a>

v-on Kısayolu

<!-- tam yazılışı -->
<a v-on:click="birŞeyYap">...</a>

<!-- kısayolu -->
<a @click="birŞeyYap">...</a>

Normal HTML’den biraz farklı görünebilirler ancak : ve @ özellik adları için geçerli karakterlerdir ve tüm Vue.js destekli tarayıcılar doğru şekilde ayrıştırılabilir. Ayrıca, son işlenen işaretlemede görünmezler. Kısayol sözdizimi tamamen isteğe bağlıdır, ancak daha sonra kullanımı hakkında daha fazla bilgi aldığınızda bunu takdir edeceksiniz.

Hesaplanmış Özellikler

Şablonlarınıza çok fazla mantık koymak, onları şişirip korumayı zorlaştırabilir.

<div id="Ornek">
{{ mesaj.split('').reverse().join('') }}
</div>

Bu noktada, şablon artık basit ve açıklayıcı değildir. Mesaj'ı ters çevirdiğini anlamadan önce bir saniyeliğine koda bakmak gerekiyor. Bu ters çevirilen mesajı bir kereden fazla görüntülemek istediğinizde sorun daha da kötüleşir.

Bu yüzden herhangi bir karmaaşık mantık için hesaplanmış özellik kullanmalısınız.

Temel Örnek

Sonuç:

Burada hesaplanmış bir özellik "reversedMessage" ilan ettik. Sağladığımız işlev, "vm.reversedMessage" özelliği için getter işlevi olarak kullanılacaktır:

Konsolu açabilir ve örnek vm ile oynayabilirsiniz. vm.reversedMessage değeri her zaman vm.message değerine bağlıdır.

Vue vm.reversedMessage’ın vm.message’a bağlı olduğunu bilir ve vm.message değiştiğinde vm.reversedMessage’a bağlı herşey değişir. En iyi yanı ise bu bağımlılık ilişkisini deklaratif olarak oluşturduğumuzdur: oluşturulan alıcı işlevinin hiçbir yan etkisi yoktur, bu da test etmeyi ve anlamayı kolaylaştırır.

Önbelleği İşleme ve Methodlar

İfadede bir yöntemi çağırıp aynı sonucu elde edebileceğimizi fark etmiş olabilirsiniz:

<p>Ters çevirilen mesaj: "{{ reversedMessage() }}"</p>
// Bileşen
metods: {
reversedMessage: function() {
return this.message.split('').reverse().join('')
}
}

İşlenmiş bir özellik yerine, aynı işlevi bir yöntem olarak tanımlayabiliriz. Sonuç olarak, iki yaklaşım aslında tamamen aynıdır. Ancak, fark, hesaplanan özelliklerin bağımlılıklarına göre önbelleğe alındığıdır. İşlenmiş bir özellik, yalnızca bazı bağımlılıkları değiştiğinde yeniden değerlendirilecektir. Burada message değişmediği sürece, reversedMessage özelliğine çoklu erişim fonksiyonu daha önce tekrar çalıştırmak zorunda kalmadan, önceden işlenen sonucu hemen döndürecektir.

Bu ayrıca, aşağıdaki işlenmiş özelliklerin hiçbir zaman güncellenmeyeceği anlamına gelir, çünkü Date.now() bir reaktif bağımlılık değildir:

compute: {
now: function() {
return Date.now()
}
}

Karşılaştırma yapıldığında, bir metot çağrısı ile her zaman yeniden işlem çalışmış olur.

Sınıf ve Stil Bağlantıları

Veri bağlamak için bir öğenin sınıf listesini ve satır içi stillerini işlemek yaygın bir ihtiyaçtır. Her ikisi de nitelik olduğundan, bunları işlemek için v-bind kullanabiliriz: ifadelerimizle yalnızca son bir dizeyi hesaplamamız gerekir.Ancak, dize birleştirme ile karıştırmak can sıkıcı ve hataya eğilimlidir. Bu nedenle vue, sınıf ve stil ile v-bind kullanıldığında özel geliştirmeler sağlar.

HTML Sınıflarını Bağlama

NesneSözdizimi

Sınıflarda dinamik olarak geçiş yapmak için bir nesneyi v-bind:class’a geçirebiliriz.

<div v-bind:class="{active: isActive }"></div>

Nesnede daha fazla alanın olmasıyla değiştirilmiş birden fazla sınıfınız olabilir. Ek olarak, v-bind:class yönergesi, düz sınıf özniteliği ile de birlikte olabilir.

<div class="static"
v-bind:class="{ active: isActive, 'text-danger': hasError }">
</div>

Ve aşağıdaki veriler:

data: {
isActive: true,
hasError: false }

Oluşacak:

<div class="static active></div>

isActive veya hasError değştiğinde sınıf listesi buna göre güncellenecektir.

<div v-bind:class=classObject></div>
data: {
classObject: {
active: true,
'text-danger': false
}
}

Bu aynı sonucu verecektir. Ayrıca, bir nesneyi döndüren hesaplanmış özelliğe de bağlanabiliriz. Bu yaygın ve güçlü bir modeldir:

<div v-bind:class="classObject"></div>
data: {
isActive: true,
error: null
},
computed: {
classObject: function() {
return {
active: this.isActive && !this.error,
'text-danger': tihs.error && this.error.type === 'fatal'
}
}
}

Dizi Sözdizimi

Sınıfları bir liste halindecode v-bind:class dizisine geçirebiliriz:

<div v-bind:class= "[activeClass, errorClass]"></div>
data: {
activeClass: 'active,
errorClass: 'text-danger'
}

Hangisi oluşacak:

<div class="active text-danger"></div>

Listede bir sınıfa da geçiş yapmak isterseniz, bunu üçlü bir ifade ile yapabilirsiniz:

<div v--bind:class="[isActive ? activeClass : '', errorClass]"></div>

errorClass her zaman geçerlidir.Ama activeClass ; isActive doğru olduğunda geçerli olacaktır.

Ancak, birden çok koşullu sınıfınız varsa bu biraz ayrıntılı olabilir. Bu nedenle, nesne sözdizimini dizi sözdiziminde kullanmak da mümkündür:

<div v-bind:class="[{ active: isActive }, errorClass]"></div>

Sonra kullanırken bazı sınıfları ekleyin:

<my-component class="baz boo"></my-component>

Oluşan HTML şöyle olacaktır:

<p class="foo bar baz boo">Merhaba</p>

Sırayla Stil Bağlama

Nesne Sözdizimi

Nesne sözdizimi v-bind:style için oldukça basittir. - Bir JavaScript nesnesi hariç, neredeyse CSS gibi görünüyor.CSS özellik adları için camelCase veya kebab-case (kebap-case ile tırnak işaretleri kullanın) kullanabilirsiniz:

<div v-bind:style="{ color: activeColor, fontSize: fontSize + 'px' }"></div>
data: {
activeColor:'Kırmızı',
fontSize: 30 }

Şablon stilinin daha net olması için stil nesnesine doğrudan bağlanması genellikle iyi bir fikirdir:

<div v-bind:style="styleObject"></div>
data: {
styleObject: {
color: 'kırmızı',
fontSize: '13px'
}
}

Yine, nesne sözdizimi genellikle nesneleri döndüren hesaplanmış özellikler ile birlikte kullanılır.

Dizi Sözdizimi

Dizi sözdizimi v-bind:style için aynı öğeye birden çok stil nesnesini uygulamanıza olanak tanır:

<div v-bind:style="[ baseStyles, overridingStyles ]"></div>

Otomatik Önek

v-bind:style‘da satıcı önekleri gerektiren bir CSS özelliğini kullandığınızda, örneğin transform, Vue, uygulanan stillere otomatik olarak uygun önekleri tespit edip ekleyecektir.

Çoklu Değerler

2.3.0+

2.3.0+ sürümünden başlayarak, bir stil özelliğine birden çok (öneki) değer dizisi sağlayabilirsiniz, örneğin:

<div v-bind:style="{ display: ['-webkit-box', '-ms-flexbox', 'flex'] }"></div>

Bu, yalnızca tarayıcının desteklediği dizideki son değeri oluşturur. Bu örnekte, flexbox'ın düzeltilmemiş sürümünü destekleyen tarayıcılar için display:flex oluşacaktır.

Koşullu Oluşum

v-if

Dize şablonlarında, örneğin Handlebar'da, böyle şartlı bir blok yazacağız:

<!-- Handlebar'ın Şablonu -->
{{ #if ok }}
<h1>Yes </h1>
{{/if}}

Vue'da, v-if yönergesini kullanarak aynı şeyi elde ederiz:


<h1 v-if="ok">Yes</h1>

V-else ile “else bloğu” eklemek de mümkündür:

<h1 v-if="ok">Yes</h1>
<h1 v-else>No</h1>

<template> / v-if ile Koşul Grupları

v-if bir yönerge ise, tek bir öğeye bağlanmalıdır. Ama ya birden fazla elemandan geçiş yapmak istersek? Bu durumda <template> öğesinde v-if kullanabiliriz, bu da görünmez bir sarıcı olarak hizmet vermektedir.Son oluşturulan sonuç <template> öğesini içermez.

<template v-if="ok">
<h1>Title </h1>
<p>Paragraf 1 </p>
<p>Paragraf 2 </p>
</template>

v-else

V-else yönergesini v-if için “else bloğ” olarak belirtmek için kullanabilirsiniz:

<div v-if="Math.random() > 0.5">
Şuan da beni görüyorsun
</div>
<div v-else>
Şuan da göremiyorsun
</div>

v-else öğesi hemen bir v-if veya v-else-if öğesini takip etmelidir - aksi halde tanınmayacaktır.

v-else-if

2.1.0+ sürümünde yeni

v-else-if isminden de anlaşılacağı gibi, v-if için “else if block” olarak kullanılır..Aynı zamanda birden çok kez zincirlenebilir:

<div v-if="type === 'A'">
A
</div> <div v-else-if="type === 'B'">
B
</div>
<div v-else-if="type === 'C'">
C
</div> <div v-else>
Not A/B/C
</div>

v-else'e benzer şekilde, bir v-else-if öğesi hemen bir v-if veya v-else-if öğesini takip etmelidir.

Yeniden Kullanılabilir Öğeleri key ile Kontrol Etme

Vue, öğeleri olabildiğince verimli hale getirmeye çalışır, genellikle sıfırdan oluşturmak yerine bunları yeniden kullanır. Vue'yi çok hızlı hale getirmenin ötesinde, bunun bazı yararlı avantajları olabilir. Örneğin, kullanıcıların birden çok oturum açma türü arasında geçiş yapmasına izin verirseniz:

<template v-if="loginType === 'username'">
<label>Username</label>
<input placeholder="Kullanıcı adınızı giriniz">
</template>
<template v-else>
<label>Email</label>
<input placeholder="Email adresinizi giriniz">
</template>

Ardından, yukarıdaki kodda loginType 'ın değiştirilmesi, kullanıcının daha önce girdiği şeyi silmez. Her iki şablon da aynı öğeleri kullandığından, <input> değiştirilmez - sadece placeholder .

Yine de bu her zaman arzu edilmez, bu yüzden Vue sizin için bir yol sunar. “Bu iki öğe tamamen ayrı - bunları tekrar kullanmayın.” Benzersiz değerlere sahip bir key özellik ekleyin:

<template v-if="loginType === 'username'">
<label>Username</label>
<input placeholder="Kullanıcı adınızı giriniz" key="username-input">
</template>
<template v-else>
<label>Email</label>
<input placeholder="Email adresinizi giriniz" key="email-input">
</template>

v-show

Bir öğeyi koşullu olarak görüntülemek için başka bir seçenek v-show yönergesidir. Kullanım büyük ölçüde aynıdır:

<h1 v-show="ok">Merhaba!</h1>

Aradaki fark şudur ki, v-show‘lu bir eleman her zaman işlenecek ve DOM'da kalacaktır; v-show yalnızca öğenin display CSS özelliğini değiştirir.

v-show'un <template> öğesini esteklemediğini ve v-else ile çalışmadığını unutmayın.

v-if vs v-show

v-if “gerçek” koşullu görüntülenir, çünkü koşul dinleyicileri içindeki olay dinleyicileri ve alt bileşenlerinin geçişler sırasında uygun şekilde imha edilmesini ve yeniden oluşturulmasını sağlar.

Karşılaştırma yapmak gerekirse v-show çok daha basittir - eleman başlangıç durumuna bakılmaksızın CSS tabanlı geçiş ile her zaman oluşturulur.

Genel olarak konuşmak gerekirse, v-if daha yüksek geçiş maliyetlerine sahipken, v-show daha yüksek başlangıç maliyetine sahiptir.Bu nedenle, sık sık bir şeyi değiştirmeniz gerekiyorsa v-show tercih edin ve eğer koşulun çalışma zamanında değişmesi olası değilse, v-if tercih edin.

v-for ile v-if

v-if ile birlikte kullanıldığında, v-for, v-if öğesinden daha yüksek bir önceliğe sahiptir. Ayrıntılar için liste oluşturma kılavuzuna bakın.

Liste İşlemleri

v-for ile Öğelere Bir Diziyi Eşleme

Bir diziye dayalı öğelerin listesini oluşturmak için v-for yönergesini kullanabiliriz. v-for yönergesi, öğelerin kaynak veri dizisi olduğu ve öğenin yinelenen dizi öğesi için bir diğer ad olduğu öğelerde ki öğe biçiminde özel bir sözdizimi gerektirir:

<ul id="example-1">
<li v-for="item in items"
{{ item.message }}
</li>
</ul>
var example1 = newVue({
data: {
items: [
{ message: 'Foo' },
{ message: 'Bar' }
]
}
})

Sonuç:

  • {{ item.message }}

v-for bloklarının içinde ana kapsam özelliklerine tam erişime sahibiz. v-for, geçerli öğenin dizini için isteğe bağlı ikinci bağımsız değişkeni de destekler.

<ul id="example-2">
<li v-for="(item, ndex) in items">
{{ parentMessage }} - {{ index}} - {{ item.message }}
</li>
</ul>
var example2 = new Vue ({
el: 'example-2',
data: {
parentMessage: 'parent',
items: [
{ message: 'Foo' },
{ message: 'Bar' }
]
}
})

Sonuç:

  • {{ parentMessage }} - {{ index }} - {{ item.message }}

ofdeğerini in yerine sınırlayıcı olarak da kullanabilirsiniz. Böylece, JavaScript'in tekrarlayıcılar için sözdizimini daha yakın olması gerekir:

<div v-for="item of items"></div>

Bir Nesne ile v-for

Bir nesnenin özellikleri boyunca yinelemek için v-for özelliğini de kullanabilirsiniz.

<ul id="v-for-object" class="demo">
<li v-for="value in object">
{{ value }}
</li>
</ul>
new Vue({
el: "#v-for-object',
data: {
object: {
isim: 'jhon',
soyisim: 'Doe',
yaş: 30
}
}
})

Sonuç:

  • {{ value }}

Anahtar için ikinci bir argüman da sağlayabilirsiniz:

<div v-for="(value, key) in object"
{{ key }}: {{ value }}
</div>
{{ key }}: {{ value }}

Ve dizin için başka bir tane:

<div v-for="(value, key, index) in object>
{{ index }}. {{ key }}: {{ value }}
</div>
{{ index }}. {{ key }}: {{ value }}

Bir nesne üzerinde yineleme yaparken,sıraJavaScript motoru uygulamalarında tutarlı olması garanti edilmeyen Object.keys() anahtar numaralandırma sırasına dayanır.

key(Anahtar)

Vue, v-for ile oluşturulan öğelerin listesini güncellerken, öntanımlı olarak “yerinde yama” stratejisini kullanır. Veri öğelerinin sırası değiştiyse, DOM öğelerini öğelerin sırasına uyacak şekilde taşımak yerine, Vue, her bir elemanı yerinde yerleştirir ve bu dizinde ne yapılması gerektiğini yansıtır. Bu, Vue 1.x'te track-by="$index" davranışına benzer.

Bu varsayılan mod verimlidir, ancak yalnızca listeniz çıktı alt bileşen durumuna veya geçici DOM durumuna (ör. Form girişi değerleri) güvenmiyorsa uygundur.

Vue'ya her düğümün kimliğini izleyebilmesi ve böylece mevcut öğeleri yeniden kullanabilmesi için bir ipucu vermek için her bir öğe için benzersiz bir key özelliği sağlamanız gerekir.Key için ideal bir değer, her bir öğenin benzersiz kimliği olabilir. Bu özel özellik 1.x'te track-by ile eşdeğerdir, ama bir öznitelik gibi çalışır, bu yüzden onu dinamik değerlere bağlamak için v-bind kullanmanız gerekir

<div v-for="item in tems" :key="item.id">
<!-- içerik -->
</div>

Yinelenen DOM içeriği basit olmadıkça veya performans kazanımları için varsayılan davranışı kasten yerine getirmediğiniz sürece, mümkün olduğunda v-for ile bir key sağlamanız önerilir.

Dizi Değişikliği Algılama

Dönüşüm Yöntemleri

Vue, gözlemlenen bir dizinin dönüşüm yöntemlerini sarar, böylece görüntüleme güncellemelerini de tetikler. Sarılmış yöntemler şunlardır:

  • push()
  • pop()
  • shift()
  • unshift()
  • splice()
  • sort()
  • reverse()

Konsolu açabilir ve önceki örneklerinitemsdizisiyle mutasyon yöntemlerini arayarak oynayabilirsiniz. Örneğin: example1.items.push({ message: 'Baz' }).

Bir Dizinin Değiştirilmesi

Değişim yöntemleri, adından da anlaşılacağı gibi, çağrılan orijinal diziyi değiştirirler. Ayrıca, karşılaştırma olarak, değişime uğramayan yöntemler de vardır, örn. Orijinal diziyi değiştirmeyen ancak her zaman yeni bir dizi döndüren filter(), concat() ve slice(), değişime uğramayan yöntemlerle çalışırken, eski diziyi yenisiyle değiştirebilirsiniz:

example1.items = example1.items.filter(function(item) {
return.item.message.match(/Foo/)
})

Bunun, şans eseri Vue'nin mevcut DOM'ı atmasına ve tüm listeyi yeniden oluşturmasına neden olacağını düşünebilirsiniz, durum böyle değil.Vue, DOM öğesinin yeniden kullanımını en üst düzeye çıkarmak için bazı akıllı sezgisel yöntemler uygular, böylece bir diziyi çakışan nesneler içeren başka bir diziyle değiştirmek çok verimli bir işlemdir.

Uyarılar

JavaScript'teki sınırlamalar nedeniyle, Vue bir dizide aşağıdaki değişiklikleri algılayamaz:

  1. Dizi ile bir öğeyi doğrudan ayarladığınızda, ör.vm.items[indexOfItem] = newValue
  2. Dizinin uzunluğunu değiştirdiğinizde, ör.vm.items.length = newLength

Örneğin;

var vm = newVue({
data: {
items: ['a', 'b, 'c']
}
})
vm.items[1] = 'x' // Tepkisizdir!
vm.items.length = 2// Tepkisizdir!

1 numaralı uyarının üstesinden gelmek için, aşağıdakilerin ikisi de vm.items[indexOfItem] = newValue ile aynı şeyi gerçekleştirecek, ayrıca reaktivite sistemindeki durum güncellemelerini de tetikleyecektir:

// Vue.set
Vue.set(vm.items, indexOfItem, newValue)
// Aray.prototype.splice
vm.items.splice(indexOfItem, 1, newValue)

Genelde Vue.set için bir takma ad olan vm.$set örneği yöntemini de kullanabilirsiniz:

vm.$set(vm.items, indexOfItem, newValue)

Uyarı 2 ile başa çıkmak için splice kullanabilirsiniz:

vm.items.splice(newLength)

Nesne Değiştirme Algılama Uyarıları

Yine, modern JavaScript'in kısıtlamaları nedeniyle, Vue özellik ekleme veya silme işlemini algılayamaz. Örneğin:

var vm = new Vue({
data: {
a: 1
}
})
//'vm.a' şimdi tepki verebilir

vm.b = 2
// 'vm.b' şimdi tepki verebilir

Vue, önceden oluşturulmuş bir örneğe dinamik olarak yeni kök düzeyi reaktif özellikler eklemesine izin vermez.Ancak, Vue.set(object, key, value) yöntemini kullanarak iç içe bir nesneye reaktif özellikler eklemek mümkündür. Örneğin, verilen:

var vm = new Vue({
data: {
userProfile: {
name: 'Anika'
}
}
})

İç içe yerleştirilmiş userProfile nesnesine yeni bir age özelliği ekleyebilirsiniz:

Vue.set(vm.userProfile, 'yas', 27)

Ayrıca,genelde Vue.set için bir takma ad olan vm.$set örnek yöntemini de kullanabilirsiniz:

vm.$set(vm.userProfile, 'yas', 27)

Bazen var olan bir nesneye yeni özellikler atamak isteyebilirsiniz, örneğin Object.assign() veya _.extend() öğesini kullanarak.Bu gibi durumlarda, her iki nesneden özelliklerle yeni bir nesne oluşturmanız gerekir. Yani yerine:

Object.assign(vm.userprofile, {
yas: 27
favoriRenk: 'Vue Yeşili'
})

Yeni, reaktif özellikler ekleyeceksiniz:

vm.userProfile = Object.assign({}, vm.userprofile, {
yas: 27
favoriRenk: 'Vue Yeşili'
})

Filtrelenmiş/Sıralı Sonuçları Görüntüle

Bazen, orijinal verileri gerçekten dönüştürmeden veya sıfırlamadan, bir dizinin filtrelenmiş veya sıralanmış bir sürümünü görüntülemek istiyoruz.Bu durumda, filtrelenmiş veya sıralanmış diziyi döndüren bir hesaplanmış özellik oluşturabilirsiniz.

Örneğin;

<li v-for="n in evenNumbers">{{ n }}</li>
data: {
numbers: [ 1, 2, 3, 4, 5]
},
computed: {
evenNumbers: function() {
return this.numbers.filter(function(number) {
return number % 2 === 0
})
}
}

Hesaplanan özelliklerin mümkün olmadığı durumlarda (ör. Iç içe geçmiş v-for döngüler), bir yöntem kullanabilirsiniz:

<li v-for="n even(numbers)">{{ n }}</li>
data: {
numbers: [ 1, 2, 3, 4, 5]
},
methods: {
even: function(numbers) {
return numbers.filter(function(number) {
return number % 2 === 0
})
}
}

Bileşen ile v-for

Bu bölüm Bileşenler hakkında bilgi sahibi olduğunu varsayar. Onu atlamaktan ve daha sonra tekrar gelmekten çekinmeyin.

Herhangi bir normal öğe gibi özel bir bileşen üzerinde doğrudan v-for kullanabilirsiniz:

<my-component v-for="item in items" :key="item.id"></my-component>
2.2.0+ sürümünde, bir bileşenle v-for kullanıldığında, şimdi bir key gereklidir.

Bununla birlikte, bu bileşenler herhangi bir veriyi otomatik olarak iletmez, çünkü bileşenlerin kendilerine ait izole edilmiş kapsamları vardır.Yinelenen verileri bileşene aktarmak için ayrıca destek kullanmalıyız:

<my-component
v-for="(item, index) in items"
v-bind:item="item"
v-bind:index="index"
v-bind:key="item.id"
></my-component>

Bileşene otomatik olarak item enjekte edilmemesinin nedeni, bileşenin v-for işlevinin nasıl çalıştığına sıkı sıkıya bağlı olmasıdır.Verilerinin nereden geldiği konusunda açık olmak, bileşeni diğer durumlarda yeniden kullanılabilir hale getirir.
