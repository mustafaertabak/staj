Vue Geliştirici Araçları

Vue kullanırken, Vue Devtools'u tarayıcınıza yüklemenizi ve Vue uygulamalarınızı daha kullanıcı dostu bir arayüzde incelemenizi ve daha kolay hata ayıklamanızı öneririz.

Doğrudan <script> Ekle

Basitçe indirin ve bir komut dosyası etiiketiyle ekleyin.Vueglobal değişken olarak kaydedilecektir.

Geliştirme sırasında küçültülmüş sürümü kullanmayın. Yaygın hatalar için tüm iyi uyarıları kaçırırsınız!


Geliştirme Versiyonu Tam uyarı ve hata ayıklama modu ile birlikte.

CDN

Manuel olarak güncelleyebileceğiniz belirli bir sürüm numarasına bağlantı oluşturmanızı öneririz:

<script src= "https://cdn.jsdelivr.net/npm/vue@2.5.16/dist/vue.js" ><script>

NPM paketinin kaynağına cdn.jsdelivr.net/npm/vue adresinden göz atabilirsiniz.

Vue'nin farklı yapılarını okuduğunuzdan ve üretim versiyonunu kullandığınızdan emin olun. Yayımlanan sitenizdeki, vue.js dosyasını vue.min.js. ile değiştirin Bu, geliştirme deneyimi yerine hız için optimize edilmiş daha küçük bir yapıdır.

NPM

Vue ile büyük ölçekli uygulamalar oluştururken NPM, önerilen kurulum yöntemidir. Webpack veya Browserify gibi modül grupları ile güzelce eşleşir. Vue ayrıca Tek Dosya Bileşenlerinin yazılması için eşlik eden araçlar sağlar.

$ npm install vue

Farklı Yapıların Tanımlanması

NPM paketinin dist/ dizininde Vue.js.'nin birçok farklı yapısını bulacaksınız.

Aralarındaki farka bir bakalım :

UMD CommmonJS ES Modülü
Tam vue.js vue.common.js vue.esm.js
Runtime-only vue.runtime.js vue.runtime.common.js vue.runtime.esm.js
Tam(Üretim) vue.min.js - -
Runtime-only(Üretim) vue.runtime.min.js - -

Terimler

  • Tam: Hem derleyiciyi hem de çalışma zamanını içeren oluşturur.

  • Derleyici: Şablon dizelerini JavaScript oluşturma işlevlerine derlemekten sorumlu kod.

  • Runtime: Sanal DOM, vb. Oluşturma ve düzeltme, Vue örneklerini oluşturmadan sorumlu kod.

  • UMD : UMD yapıları doğrudan tarayıcıda bir script etiketi ile kullanılabilir. https://cdn.jsdelivr.net/npm/vue adresindeki jsDelivr CDN'deki varsayılan dosya, Runtime + Derleyici UMD yapısıdır (vue.js).

  • CommonJS : CommonJS yapıları, browserify veya webpack 1 gibi eski gruplarla kullanım için tasarlanmıştır. Bu grupların varsayılan dosyası (pkg.main), yalnızca Çalışma Zamanı CommonJS yapısıdır(vue.runtime.common.js).

  • ES Module : ES modül yapıları, webpack 2 ya da rollup gibi modern paketlerle kullanılmak üzere tasarlanmıştır. Bu paketleyiciler için varsayılan dosya (pkg.module), yalnızca Çalışma Zamanı CommonJS yapısıdır (vue.runtime.esm.js).

Runtime+Derleyici ve Runtime-only

İstemcide şablon derlemeniz gerekiyorsa, derleyiciye ve dolayısıyla tam yapısına ihtiyacınız olacak:

// Bu derleyici gerektirir
new vue({
template: '<div>{{ hi }}</div>'
})

// Bu gerektirmez
new vue({
render (h) {
return h('div', this.hi)
}
})

vue-loader veya vueify kullanırken, *.vue dosyalarındaki şablonlar derleme zamanında JavaScript'e derlenir. Derleyiciye son pakette gerçekten ihtiyacınız yok ve bu nedenle Runtime-only kullanabilirsiniz.

Runtime-only oluşturma özellikleri, tam olarak oluşturulmuş muadillerine göre kabaca %30 daha hafif olduklarından, bunu her fırsatta kullanmalısınız. Bunun yerine tam yapıyı kullanmak istiyorsanız, paketleyicinizde bir takma ad yapılandırmanız gerekir:

Webpack

module.exports = {
//...
resolve: {
alias: {
'vue$' : 'vue/dist/vue.esm.js' // Webpack 1 için 'vue/dist/vue.common.js'
}
}
}

Rollup

const alias = require( 'toplayicinin-takma-adi' )

rollup({
//...
plugins: [
alias({
'vue' 'vue/dist/vue.esm.js'
})
]
})

Browserify

package.json'ı projenize ekleyin:

{
//...
"tarayici" : {
"vue": "vue/dist/vue.common.js"
}
}

Parcel

package.json'ı projenize ekleyin:

{
//...
"alias" : {
"vue": "./node_modules/vue/dist/vue.common.js"
}
}

Geliştirme ve Üretim Modu

Geliştirme / üretim modları UMD için sabit kodlanmıştır: minimilize edilmemiş dosyalar geliştirme içindir ve minimilize edilmiş dosyalar üretim içindir.

CommonJS ve ES Modülü yapıları, paketleyiciler için tasarlanmıştır, bu nedenle bunlar için küçültülmüş sürümler sağlarız. Son paketi kendiniz küçültmekten siz sorumlu olacaksınız.

Webpack

Webpack 4+'da, mod seçeneği kullanabilirsiniz.

module.exports={
mode:'uretim'
}

Ama Webpack 3 ve daha önceki sürümlerinde, DefinePlugin'i kullanmanız gerekir:

var webpack = require( 'webpack')

module.exports = {
//...
plugins: [
//...
new webpack.DefinePlugin({
'process.env': {
NODE_ENV: JSON .stringify( 'uretim')
}
})
}
}

Rollup

rollup-plugin-replace kullanın.

const replace = require('rollup-plugin-replace)

rollup({
//...
plugins: [
replace({
'process.env.NODE_ENV': JSON.stringify('uretim')
})
]
}).then(...)

Browserify

Global envify dönüşümünü paketinize uygulayın.

NODE_ENV=production browserify -g envify -e main.js | uglifyjs -c -m > build.js

Ayrıca bkz. Üretim Dağıtım İpuçları.

CSP ortamları

Google Chrome Uygulamaları gibi bazı ortamlarda, içeriklerin değerlendirilmesi için yeni Function() işlevinin kullanılmasını yasaklayan İçerik Güvenliği Politikası (CSP) uygulanmaktadır. Tam yapı, şablonların derlenmesi için bu özelliğe bağlıdır, bu nedenle bu ortamlarda kullanılamaz.

Öte yandan, yalnızca çalışma zamanı yapısı tamamen CSP uyumludur. Webpack + vue-loader veya Browserify + vueify ile çalışma zamanı oluşturmayı kullanırken, şablonlarınız CSP ortamlarında mükemmel şekilde çalışan render işlevleri haline getirilecektir.

Dev Yapısı

Önemli: GitHub'un /dist klasöründeki yerleşik dosyalar yalnızca sürümler sırasında kontrol edilir. GitHub'daki en yeni kaynak kodundan Vue kullanmak için, bunu kendiniz oluşturmanız gerekir!

git clone https://github.com/vuejs/vue.git node_modules/vue
cd node_modules/vue
npm install
npm run build

Bower

Sadece UMD yapıları Bower'dan temin edilebilir.

$ bower install vue
