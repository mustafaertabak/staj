Vue Geliştirici Araçları

Vue kullanırken, Vue Devtools'u tarayıcınıza yüklemenizi ve Vue uygulamalarınızı daha kullanıcı dostu bir arayüzde incelemenizi ve daha kolay hata ayıklamanızı öneririz.

CDN

Manuel olarak güncelleyebileceğiniz belirli bir sürüm numarasına bağlantı oluşturmanızı öneririz:

<script src= "https://cdn.jsdelivr.net/npm/vue@2.5.16/dist/vue.js" ><script>

NPM paketinin kaynağına cdn.jsdelivr.net/npm/vue adresinden göz atabilirsiniz.

Vue'nin farklı yapılarını okuduğunuzdan ve üretim versiyonunu kullandığınızdan emin olun. Yayımlanan sitenizdeki, vue.js dosyasını vue.min.js. ile değiştirin Bu, geliştirme deneyimi yerine hız için optimize edilmiş daha küçük bir yapıdır.

NPM

Vue ile büyük ölçekli uygulamalar oluştururken NPM, önerilen kurulum yöntemidir. Webpack veya Browserify gibi modül grupları ile güzelce eşleşir. Vue ayrıca Tek Dosya Bileşenlerinin yazılması için eşlik eden araçlar sağlar.

$ npm install vue

Farklı Yapıların Tanımlanması

NPM paketinin dist/ dizininde Vue.js.'nin birçok farklı yapısını bulacaksınız.

Aralarındaki farka bir bakalım :

UMD CommmonJS ES Modülü
Tam vue.js vue.common.js vue.esm.js
Runtime-only vue.runtime.js vue.runtime.common.js vue.runtime.esm.js
Tam(Üretim) vue.min.js - -
Runtime-only(Üretim) vue.runtime.min.js - -

Terimler

  • Tam: Hem derleyiciyi hem de çalışma zamanını içeren oluşturur.

  • Derleyici: Şablon dizelerini JavaScript oluşturma işlevlerine derlemekten sorumlu kod.

  • Runtime: Sanal DOM, vb. Oluşturma ve düzeltme, Vue örneklerini oluşturmadan sorumlu kod.

  • UMD : UMD yapıları doğrudan tarayıcıda bir script etiketi ile kullanılabilir. https://cdn.jsdelivr.net/npm/vue adresindeki jsDelivr CDN'deki varsayılan dosya, Runtime + Derleyici UMD yapısıdır (vue.js).

  • CommonJS : CommonJS yapıları, browserify veya webpack 1 gibi eski gruplarla kullanım için tasarlanmıştır. Bu grupların varsayılan dosyası (pkg.main), yalnızca Çalışma Zamanı CommonJS yapısıdır(vue.runtime.common.js).

  • ES Module : ES modül yapıları, webpack 2 ya da rollup gibi modern paketlerle kullanılmak üzere tasarlanmıştır. Bu paketleyiciler için varsayılan dosya (pkg.module), yalnızca Çalışma Zamanı CommonJS yapısıdır (vue.runtime.esm.js).

Runtime+Derleyici ve Runtime-only

İstemcide şablon derlemeniz gerekiyorsa, derleyiciye ve dolayısıyla tam yapısına ihtiyacınız olacak:

// Bu derleyici gerektirir
new vue({
template: '<div>{{ hi }}</div>'
})

// Bu gerektirmez
new vue({
render (h) {
return h('div', this.hi)
}
})

vue-loader veya vueify kullanırken, *.vue dosyalarındaki şablonlar derleme zamanında JavaScript'e derlenir. Derleyiciye son pakette gerçekten ihtiyacınız yok ve bu nedenle Runtime-only kullanabilirsiniz.

Runtime-only oluşturma özellikleri, tam olarak oluşturulmuş muadillerine göre kabaca %30 daha hafif olduklarından, bunu her fırsatta kullanmalısınız. Bunun yerine tam yapıyı kullanmak istiyorsanız, paketleyicinizde bir takma ad yapılandırmanız gerekir:

Webpack

module.exports = {
//...
resolve: {
alias: {
'vue$' : 'vue/dist/vue.esm.js' // Webpack 1 için 'vue/dist/vue.common.js'
}
}
}

Rollup

const alias = require( 'toplayicinin-takma-adi' )

rollup({
//...
plugins: [
alias({
'vue' 'vue/dist/vue.esm.js'
})
]
})

Browserify

package.json'ı projenize ekleyin:

{
//...
"tarayici" : {
"vue": "vue/dist/vue.common.js"
}
}

Parcel

package.json'ı projenize ekleyin:

{
//...
"alias" : {
"vue": "./node_modules/vue/dist/vue.common.js"
}
}

Geliştirme ve Üretim Modu

Geliştirme / üretim modları UMD için sabit kodlanmıştır: minimilize edilmemiş dosyalar geliştirme içindir ve minimilize edilmiş dosyalar üretim içindir.

CommonJS ve ES Modülü yapıları, paketleyiciler için tasarlanmıştır, bu nedenle bunlar için küçültülmüş sürümler sağlarız. Son paketi kendiniz küçültmekten siz sorumlu olacaksınız.

Webpack

Webpack 4+'da, mod seçeneği kullanabilirsiniz.

module.exports={
mode:'uretim'
}

Ama Webpack 3 ve daha önceki sürümlerinde, DefinePlugin'i kullanmanız gerekir:

var webpack = require( 'webpack')

module.exports = {
//...
plugins: [
//...
new webpack.DefinePlugin({
'process.env': {
NODE_ENV: JSON .stringify( 'uretim')
}
})
}
}

Rollup

rollup-plugin-replace kullanın.

const replace = require('rollup-plugin-replace)

rollup({
//...
plugins: [
replace({
'process.env.NODE_ENV': JSON.stringify('uretim')
})
]
}).then(...)

Browserify

Global envify dönüşümünü paketinize uygulayın.

NODE_ENV=production browserify -g envify -e main.js | uglifyjs -c -m > build.js

Ayrıca bkz. Üretim Dağıtım İpuçları.

CSP ortamları

Google Chrome Uygulamaları gibi bazı ortamlarda, içeriklerin değerlendirilmesi için yeni Function() işlevinin kullanılmasını yasaklayan İçerik Güvenliği Politikası (CSP) uygulanmaktadır. Tam yapı, şablonların derlenmesi için bu özelliğe bağlıdır, bu nedenle bu ortamlarda kullanılamaz.

Öte yandan, yalnızca çalışma zamanı yapısı tamamen CSP uyumludur. Webpack + vue-loader veya Browserify + vueify ile çalışma zamanı oluşturmayı kullanırken, şablonlarınız CSP ortamlarında mükemmel şekilde çalışan render işlevleri haline getirilecektir.

Giriş

Vue.js Nedir?

Vue(teleffuz edilişi /vju:/, view gibi), kullanıcı arayüzleri oluşturmak için ilerici bir yapıdır. Diğer yapılardan farklı olarak Vue, aşamalı olarak kabul edilebilir hale gelecek şekilde tasarlanmıştır.Kütüphanenin çekirdeği sadece görünüm katmanına odaklanır ve diğer kütüphanelerle veya mevcut projelerle toplayıp bütünleştirilmesi kolaydır. Öte yandan, Vue modern araçlar ve destekleyici kütüphanelerle birlikte kullanıldığında sofistike Tek Sayfa Uygulamalarına da mükemmel bir şekilde güç sağlayabilir.

Tecrübeli bir ön geliştirici iseniz ve Vue'nin diğer kütüphaneler / çerçeveler ile nasıl karşılaştırıldığını öğrenmek istiyorsanız, Diğer Çerçevelerle Karşılaştırma'ya göz atın.

İşlem Tanımlama

Bu dersi Scrimba'da dene

Vue.js’in çekirdeği, basit şablon sözdizimini kullanarak DOM’daki verilere bildirimde bulunmamızı sağlayan bir sistemdir:

<div id="app">
{{ mesaj }}
</div>
var app = new Vue({
el: '#app' ,
data: {
mesaj: 'Merhaba Vue!'
}
})
{{ message }}

İlk Vue uygulamamızı oluşturduk! Bu, bir dize şablonu oluşturmaya benziyor, ancak vue başlık altında çok fazla iş yaptı. Veriler ve DOM artık birbirine bağlı ve herşey artık duyarlı.Nasıl bilebiliriz? Tarayıcınızın JavaScript konsolunu (hemen bu sayfada) açın ve app.message değerini farklı bir değere ayarlayın. Yukarıdaki güncellemeye göre hazırlanan örneği görmelisiniz.

Metnin ara değerlerine ek olarak, bunun gibi öğe özelliklerini de bağlayabiliriz:

<div id="app-2">
<span v-bind:title="message">
Fareyi birkaç sanyeliğine burada tutarsanız başlığı görebilirsiniz!
</span>
</div>
var app2 = new Vue ({
el: '#app-2',
data: {
message: 'Bu sayfayı + new Date().toLocaleString() + 'yüklediniz!'
}
})
Fareyi birkaç saniyeliğine burada tutarsanız başlığı görebilirsiniz!

Burada yeni bir şeyle karşılaşıyoruz. Gördüğünüz v-bind özelliğine direktif denir. Direktifler, Vue tarafından sağlanan özel nitelikler olduklarını belirtmek için v- ile öne eklenir ve tahmin edebileceğiniz gibi, oluşturulan DOM’a özel duyarlı davranışlar uygularlar. Burada temel olarak “Bu öğenin title (başlık) özelliğini Vue örneğindeki mesaj özelliği ile güncel tut” deyip duruyor.

JavaScript konsolunuzu tekrar açarsanız app2.message = 'yeni mesaj', bir kez daha bağlı HTML'nin - bu durumda title(başlık) özelliğinin - güncellendiğini göreceksiniz.

Kosullar ve Döngüler

Bu dersi Scrimba'da dene

Bir öğenin görünürlüğünü değiştirmek kolaydır.

<div id="app-3">
<span v-if="seen">Şimdi Beni Görüyorsun </span>
</div>
var app3 = new Vue({
el: '#app-3',
data: {
seen: true
}
})
Şimdi Beni Görüyorsun

Devam edin ve konsola app3.seen = false değerini girin. Mesajın kaybolduğunu göreceksiniz.

Bu örnek, verileri yalnızca metin ve niteliklere değil, aynı zaman da DOM'ın yapısına da bağlayabilceğimizi göstermektedir. Ayrıca, Vue tarafndan öğeler eklendiğinde/ güncellendiğinde/kaldırıldığında geçiş efektlerini otomatik olarak uygulayabilen güçlü bir geçiş efekti sitemi sağlar.

Her biri kendi özel işlevselliğine sahip bir kaç direktif daha var. Örneğin, v-for yönergesi kullanılarak, bir dizideki öğelerin listesi görüntülenebilir.

<div id="app-4">
<ol>
<li v-for="todo in todos">
{{ todo.text }}
</li>
</ol>
</div>
var app4 = new Vue({
el: '#app-4',
data: {
todos: [
{ text: 'Javascript'i öğren' },
{ text: 'Vue'yi öğren' },
{ text: 'Harika bir şey inşa et' }
]
}
})
  1. {{ todo.text }}

Konsolda, app4.todos.push({text:’Yeni nesne’}) giriniz. Listeye eklenen yeni öğeyi görüceksiniz.

Kullanıcı Girişini Kullanma

Bu dersi Scrimba'da dene

Kullanıcıların uygulamanızla etkileşime geçmesine izin vermek için, Vue örneklerimizde yöntemleri çağırmaya yönelik etkinlik dinleyicileri eklemek için v-on yönergesini kullanabiliriz:

<div id="app-5">
<p>{{ message }} </p>
<button v-on:click="reverseMessage">Mesajı Ters Yaz</button>
</div>
var app5 = new Vue({
el: '#app-5',
data: {
message: 'Merhaba Vue.js'
},
methods: {
reverseMessage: function () {
this.message = this.message.split('').reverse().join('')
}
}
})

{{ message }}

Bu yöntemde, DOM'a dokunmadan uygulamanızın durumunu güncelleriz. Tüm DOM manipülasyonları Vue tarafından işlenir ve yazdığınız kod temel mantığa odaklanır.

Vue ayrıca, form girişi ve uygulama arasında iki yönlü bir bağlantı kuran v-model yönergesini de sağlar:

<div id="app-6">
<p>{{ message }} </p>
<input v-model="message">
</div>
var app6 = new Vue({
el: '#app-6',
data: {
message: 'Merhaba Vue!'
}
})

{{ message }}

Bileşenler Oluşturma

Bu dersi Scrimba'da dene

Bileşen sistemi, Vue'daki diğer önemli bir kavramdır, çünkü küçük, kendi kendine yeten ve çoğu zaman yeniden kullanılabilen bileşenlerden oluşan büyük ölçekli uygulamalar oluşturmamıza olanak veren bir soyutlamadır. Bunu düşünürsek, hemen hemen her türlü uygulama arayüzü, bileşen ağacına ayrılabilir:

Vue'da, bir bileşen esas olarak önceden tanımlanmış seçeneklere sahip bir Vue örneğidir. Bir bileşenin Vue'ya kaydedilmesi basittir:

//Todo-item adı verilen yeni bir bileşen tanımlayın
Vue.component('todo-item', {
template: '<li>Bu bir todo</li>'
})

Şimdi başka bir bileşenin şablonunda yazabilirsiniz:

<ol>
<!--todo-item bileşeni örneğin oluşturma -->
<todo-item></todo-item>
</ol>

Ancak bu her bir todo için aynı metni oluşturur, bu gayet normal. Ana kapsamdaki verileri çocuk bileşenlerine geçirebilmeliyiz. Bir prop kabul etmesini sağlamak için bileşen tanımını değiştirelim:

Vue.component('todo-item', {
//todo-item bileşeni artık prop'u bir özel özellik
// gibi kabul eder.
//Bu prop'a todo denir.
props: ['todo'],
template: '<li>{{ todo.text }}</li>'
})

Şimdi v-bind kullanarak her tekrarlanan bileşene todo geçiş yapabiliriz:

<div id="app-7">
<ol>
<!--
Şimdi her todo-item todo nesnesini temsil ediyor, böylece içeriği
Dinamik oluyor.Ayrıca her bir bileşeni bir “anahtar" ile
sağlamamız gerekiyor.
-->
<todo-item
v-for="item in groceryList"
v-bind:todo="item"
v-bind:key="item.id">
&/todo-item>
</ol>
</div>
Vue.component('todo-item', {
props: ['todo'],
template: '<li>{{ todo.text }}</li>'
})

var app7 = new Vue({
el: '#app-7',
data: {
groceryList: [
{ id: 0, text: 'Sebzeler' },
{ id: 1, text: 'Peynir' },
{ id: 2, text: 'İnsanlar başka ne yemeli?' },
]
}
})

Bu, zor bir örnektir, ancak uygulamamızı iki küçük bölüme ayırmayı başardık ve çocuk, props arabirimi aracılığıyla ebeveynden makul ölçüde iyi bir şekilde ayrıştırıldı.

Büyük bir uygulamada, tüm uygulamayı geliştirirken yönetimi kolaylaştırmak için bileşenlere bölmek gerekir. İleride bileşenler hakkında çok daha fazla konuşacağız, ancak bir uygulamanın şablonunun bileşenlerle neye benzeyebileceğne dair bir örnek:

<div id="app">
<app-nav></app-nav>
<app-view>
<app-sidebar></app-sidebar>
<app-content></app-content>
</app-view>
</div>

Vue Örneği

Bir Vue Örneği Oluşturmak

Her bir Vue uygulaması, Vue işlevine sahip yeni bir Vue örneği oluşturarak başlar:

var vm = new Vue({
//ayarlar
})

MVVM modeli ile kesinlikle ilişkili olmamasına rağmen, Vue’nin tasarımı kısmen esinlenmiştir. Bir kural olarak, Vue örneğimize başvurmak için genellikle vm değişkenini (ViewModel kısaltılması) kullanırız.

Bir Vue uygulaması, isteğe bağlı olarak yuvalanmış, yeniden kullanılabilir bileşenlerin bir ağacına düzenlenmiş yeni Vue ile oluşturulan bir kök Vue örneğinden oluşur. Örneğin, bir yapılacaklar uygulamasının bileşen ağacı şöyle görünebilir:

Ana Yapı
└─ YapılacaklarListesi
├─ Yapılacak İş
│ ├─ GöreviSilButonu
│ └─ GörevGüncelleButonu
└─ YapılacaklarListesiAltBilgi
├─ YapılacaklarıTemizleButonu
└─ YapılacaklarListesiİstatikleri

Veri ve Methodlar

Bir Vue örneği oluşturulduğunda, veri nesnesinde bulunan tüm özellikleri Vue’nin reaktivite sistemine ekler. Bu özelliklerin değerleri değiştiğinde, görünüm “yeni değerler” ile eşleşecek şekilde güncellenir.

//veri nesnemiz
var veri = { a: 1 }

// Nesne bir vue örneğine eklenir
var vm = new Vue({
veri: veri
})

// Örnekte özellik edinme
// orjinal verilerden birini döndürür
vm.a == veri.a // => true
// Örnekte özelliğin değiştirilmesi
// orjinal verileri de etkiler
vm.a = 2
data.a // => 2

//... ve tersi içinde geçerli.
data.a = 3
vm.a // => 3

Bu veriler değiştiğinde, görünüm yeniden oluşturulur. Verilerdeki özelliklerin, yalnızca örnek oluşturulduğunda mevcutsa reaktif olduklarına dikkat edilmelidir. Bu, aşağıdaki gibi yeni bir özellik eklerseniz:

vm.b = 'Merhaba'

Ardından b olarak yapılan değişiklikler, herhangi bir görünüm güncellemesini tetiklemez. Daha sonra bir özelliğe ihtiyacınız olacağını biliyorsanız, bazı başlangıç değerlerini ayarlamanız gerekir. Örneğin :

veri: {
yeniYapılacakİş: '',
görüntülenmeSayısı: 0,
tamamlanmışTodolarıGizle: false,
yapılacaklar: [],
error: null
}

Bunun tek istisnası, mevcut özelliklerin değiştirilmesini engelleyen Object.freeze() kullanımıdır, bu da reaktivite sisteminin değişiklikleri izleyemediği anlamına gelir.

var obj = {
foo: 'bar'
}

Object.freeze(obj)

new Vue({
el: '#app',
veri: obj
})
<div id="app">
<p>{{ foo }} </p>
<!-- Bu artık foo'yu güncelleyemez! -->
<button v-on:click="foo = 'baz'">Değiştir</button>
</div>

Veri özelliklerine ek olarak, Vue örnekleri bir dizi yararlı örnek özellik ve yöntemi açığa çıkarır. Bunlar, kullanıcı tanımlı özelliklerden ayırt etmek için $ ile öneklenir. Örneğin:

var data = { a: 1}
var vm = new Vue({
el: '#ornek',
veri: veri
})

vm.$veri === veri // => true
vm.$el === document.getElementById('ornek') // => true

// $izle bir örnek methodtur
vm.$izle('a', function) {
// vm.a değiştiğinde bu geri arama çağrılır.
})

Örnek Akış Kancaları

Her bir Vue örneği, oluşturulduğunda bir dizi başlatma adımından geçer - örneğin, veri gözlemini kurması, şablonu derlemesi, örneği DOM'a bağlaması ve veri değiştiğinde DOM'ı güncellemesi gerekir. Ayrıca yaşam döngüsü kancaları olarak adlandırılan fonksiyonları çalıştırır ve yol boyunca kullanıcılara belirli aşamalarda kendi kodlarını ekleme fırsatı verir.

Örneğin oluşturulan kanca, bir örnek oluşturulduktan sonra kodu çalıştırmak için kullanılabilir:

new Vue({
veri: {
a: 1
},
olusturulan: function() {
// bu, vm örneğini işaret ediyor.
console.log('a =' + this.a)
}
})
// => "a = 1"

Eklenen, güncellenen ve yok edilen yaşam d öngüsü örneğinin farklı aşamalarında çağrılacak başka kancalar da vardır. Tüm yaşam döngüsü kancaları, bu bağlamı çağırmak için Vue örneğini işaret ederek bu içeriğe çağrılır.

Hedef fonksiyonlarını, bir ayarlar özelliğinde veya geri aramada, örneğin: created: () => console.log(this.a) veya vm.$watch('a', newValue => this.myMethod()) gibi fonksiyonlarda kullanmayın. Hedef fonksiyonları ana içeriiğe bağlı olduğunan, bu beklediğiniz gibi bir Vue örneği olmaz, genellikle Uncaught TypeError: Cannot read property of undefined ya da Uncaught TypeError: this.myMethod is not a function gibi hatalarla sonuçlanır.

Şablon Söz Dizimi

Vue.js, oluşturulmuş DOM'yi temel Vue örneğinin verilerine bildirmenize olanak tanıyan bir HTML tabanlı şablon sözdizimi kullanır. Tüm Vue.js şablonları, uyumlu tarayıcılar ve HTML ayrıştırıcıları tarafından ayrıştırılabilen geçerli bir HTML'dir.

Virtual DOM kavramlarına aşina iseniz ve JavaScript'in ham gücünü tercih ediyorsanız, isteğe bağlı JSX desteği ile şablonlar yerine render işlevlerini doğrudan yazabilirsiniz.

Ara Değer Bulma

Metin

En basit veri bağlama biçimi “Bıyık” sözdizimini (çift küme parantezleri) kullanarak metin enterpolasyonudur:

<span>Message: {{ msg }}</span>

Bıyık etiketi, ilgili veri nesnesi üzerindeki msg özelliğinin değeriyle değiştirilecektir. Ayrıca, veri nesnesinin msg özelliği değiştiğinde de güncellenecektir.

Aynı zamanda v-once direktifi kullanarak veri değişiminde güncellenmeyen bir kerelik ara değerleri de gerçekleştirebilirsiniz, ancak aynı alandaki diğer bağlamaları da etkileyeceğini unutmayın:

<span v-once>Bu asla değişmeyecek: {{ msg }}</span>

Ham HTML

Çift süslü parantez, verileri HTML değil düz metin olarak yorumlar. Gerçek HTML çıktısı almak için v-html yönergesini kullanmanız gerekir:

<p>Süslü parantez kullanma: {{ hamHTML }}</p>
<p>v-html direktifini kullanma: <span v-html="hamHTML"</p>

Süslü parantez kullanma: {{ hamHTML }}

v-html direktifi kullanma:

span içeriği, düz HTML olarak yorumlanan rawHtml özelliğinin değeriyle değiştirilir; veri bağlantıları yoksayılır. Şablon kısmi oluşturmak için v-html kullanamayacağınızı unutmayın, çünkü Vue, dize tabanlı bir şablon oluşturma motoru değildir.

Web sitenizde dinamik HTML'yi dinamik olarak oluşturmak çok tehlikeli olabilir çünkü XSS güvenlik açıklarına kolayca yol açabilir.Yalnızca güvenilir içerik üzerinde HTML interpolasyonunu kullanın ve asla kullanıcı tarafından sağlanan içerik üzerinde kullanmayın.

Özellikler

Süslü parantezler HTML özniteliklerinde kullanılamaz. Bunun yerine bir v-bind direktifini kullanın:

<div v-bind:id="dinamikId"></div>

Sadece varlıkların doğru olduğu yerlerde, boole özelliklerinde, v-bind biraz farklı çalışır. Bu örnekte:

<button v-bin:disabled="butonDevredisi">Button</div>

Eğer butonDevredisi öğesi null, undefined veya false değerine sahipse, disabled özelliği oluşturulmuş <button> öğesine dahil edilmez.

Javascript İfadelerini Kullanma

Şimdiye kadar sadece şablonlarımızdaki basit özellik anahtarlarına bağlanıyoruz. Ancak Vue.js aslında tüm veri bağlamaları içindeki JavaScript ifadelerinin tam gücünü desteklemektedir:

{{ numara + 1 }}

{{ tamam ? 'EVET' : 'HAYIR' }}

{{ message.split('').reverse().join('') }}

<div v-bind:id="'list-' + id"></div>

Şablon ifadeleri korumalı ve yalnızca Matematik ve Tarih gibi bir beyaz küre listesine erişebilir. Şablon ifadelerinde kullanıcı tanımlı globallere erişmeyi denememelisiniz.

Direktifler

Talimatlar v- ön eki ile ayrıcalıklı özelliklerdir. Talimat özellik değerlerinin tek bir JavaScript ifadesi olması beklenir (v-for için istisna, daha sonra tartışılacaktır).Bir direktörün görevi, ifade ettiği değer değiştiğinde, DOM’a yan etkileri tepkisel olarak uygulamaktır.Girişte gördüğümüz örneği inceleyelim:

<p v-if="gor">Şimdi beni görüyorsun.</p>

Burada, v-if yönergesi gor ifadesinin değerinin doğruluğuna dayanarak <p> öğesini kaldıracak / ekleyecektir.

Argümanlar

Bazı direktifler "argüman" alabilirler ve direktif adından sonra bir kolon tarafndan gösterilirler. Örneğin, v-bind yönergesi bir HTML özniteliğini yeniden etkinleştirmek için kullanılır:

<a v-bind:href="url">...</a>

Burada href, v-bind yönergesinin, öğenin href özniteliğini ifade url'sinin değerine bağladığını söyleyen argümandır.

Bu örnek ise DOM olaylarını dinleyen v-on yönergesidir:

<a v-on:click="birŞeyYap">...</a>

Düzenleyiciler

Değiştiriciler, bir nokta ile belirtilen özel postfix'lerdir. Bir direktifin bazı özel yollarla bağlanması gerektiğini belirtir.Örneğin, .prevent değiştiricisi, tetiklenen olayla ilgili v-on yönergesine event.preventDefault() öğesini çağırmasını söyler:

<form v-on:submit.prevent="Gonder">...</form>

Kısayollar

v- ön eki, şablonlarınızdaki Vue'ya özgü öznitelikleri tanımlamak için görsel bir işaret görevi görür. Bu, varolan bazı biçimlendirmelere dinamik davranış uygulamak için Vue.js'yi kullanırken yararlıdır. ama sıkça kullanılan bazı direktifler için ayrıntılı olabilir. Aynı zamanda Vue.js'nin her şablonu yönetdiği bir SPA oluştururken v- öneki gereksinimi daha az önem kazanır. Bu nedenle, Vue.js en sık kullanılan direktiflerden ikisi, v-bind ve v-on için özel kısayollar sağlar:

v-bind Kısayolu

<!-- tam yazılışı -->
<a v-bind:href="url">...</a>

<!-- kısayolu -->
<a :href="url">...</a>

v-on Kısayolu

<!-- tam yazılışı -->
<a v-on:click="birŞeyYap">...</a>

<!-- kısayolu -->
<a @click="birŞeyYap">...</a>

Normal HTML’den biraz farklı görünebilirler ancak : ve @ özellik adları için geçerli karakterlerdir ve tüm Vue.js destekli tarayıcılar doğru şekilde ayrıştırılabilir. Ayrıca, son işlenen işaretlemede görünmezler. Kısayol sözdizimi tamamen isteğe bağlıdır, ancak daha sonra kullanımı hakkında daha fazla bilgi aldığınızda bunu takdir edeceksiniz.
